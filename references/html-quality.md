# HTML Quality Rules

The showcase is a reading and discovery artifact, not a generic dashboard.

- One project tab per source collection; keep tab selection keyboard accessible.
- Search across skill name, purpose, capability, input, output, and examples.
- Show result counts and a useful empty state.
- Put the source link beside each skill and the repository link in the project header.
- Use specific section names: “What it does”, “What you provide”, “What you get”, and “Try asking”.
- Prefer restrained editorial styling, readable density, and strong hierarchy over gradients, glass effects, or decorative cards.
- Keep the page usable at 375 px without document-level horizontal overflow.
- Escape all source data and render dynamic strings with `textContent`.
- Make the final page self-contained and do not include absolute local paths.
- Verify language switching, project tabs, search, focus behavior, and the browser console.
